2. How would you track down a performance issue in production? Have you ever had to do this?

To track down a performance issue in production, I would:

1. Identify the affected page or API.
2. Use browser developer tools to inspect network requests.
3. Check server logs and application logs.
4. Analyze database queries for slow execution.
5. Monitor CPU, memory, and network usage.
6. Optimize slow queries, unnecessary API calls, or large assets.

As a frontend developer, I have investigated slow-loading pages by analyzing API response times, reducing unnecessary re-renders, and optimizing component rendering.
